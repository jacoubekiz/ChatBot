ACL Results API Documentation

Base URL

https: //api.acl.com/lab/v1

Authentication

All requests require API key authentication:

Authorization: Bearer {your_api_key
}

Endpoint

Get Patient Details with Results URL

Method: GET

Endpoint: /patient/results

Parameters

Parameter Type Description Example
mobile string Patient's mobile number with country code (required) 966501234567
dob string Date of birth for verification (YYYY-MM-DD, optional) 1985-03-15

Example Request

curl -X GET \
  'https: //api.acl.com/lab/v1/patient/results?mobile=966501234567' \
  -H 'Authorization: Bearer your_api_key_here'

Response

Success Response

{
    "success": true,
    "data": {
        "patientId": "PAT123456",
        "mobile": "96650**67",
        "name": "Ahmed Mohammed",
        "resultsUrl": "https://api.acl.com/lab/v1/results/PAT123456",
        "lastTestDate": "2023-11-15",
        "availableResults": 3
    }
}

Response Fields

Field Type Description
patientId string Unique patient identifier
mobile string Masked mobile number for privacy
name string Patient's full name
resultsUrl string URL to access detailed results
lastTestDate string Date of most recent test (YYYY-MM-DD)
availableResults integer Number of available test results

Error Responses

Code Response Description
400 
{
    "success": false,
    "error": "INVALID_MOBILE_FORMAT"
}
 Invalid mobile number format
404 
{
    "success": false,
    "error": "PATIENT_NOT_FOUND"
}
 No patient found with this mobile number
401 
{
    "success": false,
    "error": "UNAUTHORIZED"
}
 Invalid or missing API key

Security Notes

  • Mobile numbers are partially masked in responses
  • All requests require valid API key
  • Results URLs contain unique identifiers instead of mobile numbers
  • Always use HTTPS for all requests
  • Recommended to implement rate limiting (100 requests/minute)
